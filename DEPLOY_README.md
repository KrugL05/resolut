## 1. Что и куда выкладывать

Проект состоит из двух частей:

```
resolute/
├── public/           ← ЭТО выкладывается на хостинг (сайт)
├── netlify/          ← ЭТО нужно только для Netlify (серверная функция)
│   └── functions/
│       └── submit-form.js
├── netlify.toml      ← ЭТО нужно только для Netlify (конфиг)
└── backend/          ← ЭТО документация и скрипт Sheets, НЕ выкладывается
```

> ⚠️ Папка `backend/` на хостинг **не загружается** — там только инструкции
> и Google Apps Script, который настраивается отдельно.

## 2. Вариант A — Netlify (бесплатно)

**Что даёт:** HTTPS, CDN, серверные функции (нужны для скрытия токена бота),
100 ГБ трафика в месяц. Карта не нужна.

### Способ 1 — Перетащить папку (проще всего, 2 минуты)

> ⚠️ Этот способ **не поддерживает** серверные функции (Netlify Functions).
> Форма будет работать, но токен Telegram будет виден в исходнике.
> Подходит для быстрого старта, но не рекомендуется для продакшна.

1. Зайдите на [netlify.com](https://netlify.com) → Sign up (через GitHub или email)
2. На главной странице найдите зону **"Want to deploy a new site without connecting to Git?"**
3. Перетащите туда папку **`public/`** (не весь архив, именно папку)
4. Netlify автоматически даст адрес вида `random-name-123.netlify.app`
5. Можно переименовать: Site settings → Site name → Change site name

---

### Способ 2 — Через GitHub (рекомендуется, Functions работают)

Этот способ подключает весь проект включая серверную функцию —
токен Telegram будет надёжно скрыт на сервере.

#### Шаг 1 — Загрузить проект на GitHub

1. Зарегистрируйтесь на [github.com](https://github.com)
2. Нажмите **New repository** → название `resolute-website` → Create
3. Загрузите файлы:
   - Нажмите **uploading an existing file**
   - Перетащите **всё содержимое** архива (папки `public/`, `netlify/`,
     файлы `netlify.toml`, `README.md`)
   - Нажмите **Commit changes**

#### Шаг 2 — Подключить к Netlify

1. Зайдите на [netlify.com](https://netlify.com) → **Add new site → Import an existing project**
2. Выберите **GitHub** → авторизуйте → выберите репозиторий `resolute-website`
3. Настройки сборки заполнятся автоматически из `netlify.toml`:
   - Build command: *(оставьте пустым)*
   - Publish directory: `public`
4. Нажмите **Deploy site**
5. Дождитесь зелёного статуса **Published** (30–60 секунд)

#### Шаг 3 — Добавить переменные окружения (токены)

Это самый важный шаг — именно здесь токены хранятся безопасно.

1. В Netlify: **Site settings → Environment variables → Add a variable**
2. Добавьте три переменные:

| Key | Value |
|-----|-------|
| `TELEGRAM_BOT_TOKEN` | `7123456789:AAFxxxxxxxxx` (ваш токен от BotFather) |
| `TELEGRAM_CHAT_ID` | `1054823888` (ваш Chat ID) |
| `SHEETS_WEBHOOK_URL` | `https://script.google.com/macros/s/.../exec` (из шага 5) |

3. После добавления переменных: **Deploys → Trigger deploy → Deploy site**
   (нужно передеплоить, чтобы функция подхватила переменные)

#### Шаг 4 — Проверить

1. Откройте сайт по адресу вида `resolute.netlify.app`
2. Заполните форму → нажмите «Отправить заявку»
3. В Telegram должно прийти сообщение с данными заявки

#### Подключить свой домен (опционально)

1. Купите домен, например `resolute-cheer.ru` на [reg.ru](https://reg.ru) (~200₽/год)
2. В Netlify: **Domain settings → Add custom domain**
3. Введите домен → Netlify покажет DNS-записи
4. В панели reg.ru: DNS → добавьте записи от Netlify
5. HTTPS подключится автоматически через 10–30 минут

## 3. Вариант B — Российский хостинг

Подходит если нужны: рублёвая оплата, российская поддержка, домен .ru.

> ⚠️ Обычный хостинг (виртуальный) **не поддерживает** Node.js функции.
> Это значит, что `netlify/functions/submit-form.js` работать **не будет**.
> Для работы формы нужно будет указать токены прямо в `public/js/form.js`
> (менее безопасно) или взять VPS (дороже, но полный контроль).

### Вариант B1 — Виртуальный хостинг (Beget / TimeWeb)

**Beget:** от 99₽/месяц — [beget.com](https://beget.com)
**TimeWeb:** от 99₽/месяц — [timeweb.com](https://timeweb.com)

#### Что сделать перед загрузкой

Откройте `public/js/form.js` и временно пропишите токены напрямую
(до тех пор, пока не будет нормального бэкенда):

```js
// Найдите строки с fetch и замените URL функции на прямые вызовы API
// Подробнее — в файле backend/README.md, раздел "Режимы работы"
```

#### Загрузка через Файловый менеджер (проще)

1. Зайдите в панель управления хостингом (cPanel или ISPmanager)
2. Откройте **Файловый менеджер**
3. Перейдите в папку `public_html/`
4. Удалите стандартный `index.html` если есть
5. Нажмите **Загрузить** → загрузите весь архив → распакуйте
6. Убедитесь, что ваш `index.html` лежит прямо в `public_html/`, а не в подпапке

#### Загрузка через FTP (для тех, кто знает)

Данные для FTP: в панели хостинга → FTP-аккаунты

```
Хост:     ftp.ваш-домен.ru
Логин:    u1234567
Пароль:   ваш пароль
Порт:     21
```

Программы: [FileZilla](https://filezilla-project.org/) (бесплатно)

Загрузите содержимое папки `public/` → в папку `public_html/` на сервере.

#### Подключить домен

На Beget и TimeWeb домен можно купить прямо в панели управления.
При регистрации часто дают домен в подарок на первый год.

### Вариант B2 — VPS (полный контроль, Node.js работает)

Подходит если нужны серверные функции и максимальная гибкость.

**Провайдеры:** Beget VPS (от 299₽/мес), TimeWeb Cloud (от 259₽/мес),
Selectel (от 350₽/мес), Yandex Cloud.

#### Минимальная настройка VPS (Ubuntu 22.04)

```bash
# 1. Подключитесь по SSH
ssh root@ВАШ_IP

# 2. Установите Node.js
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs

# 3. Установите PM2 (менеджер процессов)
npm install -g pm2

# 4. Установите Nginx (веб-сервер)
apt install -y nginx

# 5. Скопируйте проект на сервер (с вашего компьютера)
scp -r ./resolute root@ВАШ_IP:/var/www/resolute

# 6. Создайте простой Express-сервер вместо Netlify Functions
# (обратитесь к backend/README.md за полным кодом)

# 7. Запустите через PM2
pm2 start server.js --name resolute
pm2 save
pm2 startup
```

## 4. Настройка Telegram Bot

### Шаг 1 — Создать бота

1. Откройте Telegram, найдите **@BotFather**
2. Напишите: `/newbot`
3. Введите название: `Resolute Club`
4. Введите username: `resolute_club_bot` (должен быть уникальным)
5. Скопируйте **токен**: `7123456789:AAFxxxxxxxxxxxxxx`

### Шаг 2 — Получить Chat ID

1. Напишите своему боту любое сообщение (например: `привет`)
2. Откройте в браузере (вставьте свой токен):
   ```
   https://api.telegram.org/bot7123456789:AAFxxxx/getUpdates
   ```
3. В ответе найдите:
   ```json
   "chat": { "id": 1054823888 }
   ```
4. Число `1054823888` — это ваш **Chat ID**

> **Совет:** Создайте отдельную Telegram-группу «RESOLUTE Заявки»,
> добавьте туда бота и сделайте его администратором.
> Тогда все менеджеры смогут видеть заявки в одном месте.
> Chat ID группы будет отрицательным: `-100XXXXXXXXX`

### Шаг 3 — Проверить (вставьте свои TOKEN и CHAT_ID)

Откройте в браузере:
```
https://api.telegram.org/botВАШ_TOKEN/sendMessage?chat_id=ВАШ_CHAT_ID&text=Тест
```
Если пришло сообщение «Тест» — всё работает.

## 5. Настройка Google Sheets

### Шаг 1 — Создать таблицу

1. Откройте [sheets.google.com](https://sheets.google.com)
2. Создайте новую таблицу → назовите **«RESOLUTE — Заявки»**

### Шаг 2 — Вставить скрипт

1. В таблице: **Расширения → Apps Script**
2. Удалите весь код в редакторе
3. Откройте файл `backend/google-apps-script.gs` из архива
4. Скопируйте его содержимое и вставьте в редактор
5. Нажмите 💾 **Сохранить проект**

### Шаг 3 — Опубликовать

1. Нажмите **Развернуть → Новое развёртывание**
2. Тип: **Веб-приложение** (нажмите на шестерёнку рядом с «Выбрать тип»)
3. Описание: `v1`
4. Выполнять от имени: **Я**
5. Кто имеет доступ: **Все**
6. Нажмите **Развернуть**
7. При первом деплое — нажмите **Разрешить доступ** → выберите свой аккаунт →
   нажмите «Дополнительно» → «Перейти на страницу (небезопасно)» → Разрешить
8. Скопируйте **URL веб-приложения**:
   ```
   https://script.google.com/macros/s/AKfycbx.../exec
   ```

### Шаг 4 — Вставить URL

- **Netlify:** добавьте в Environment variables как `SHEETS_WEBHOOK_URL`
- **Обычный хостинг:** вставьте URL в `public/js/form.js` в переменную CONFIG

### Как выглядит таблица после первой заявки

| Дата/Время | Имя ребёнка | Возраст | Родитель | Телефон | Время | Коммент | Статус |
|---|---|---|---|---|---|---|---|
| 19.05.2025 14:30 | Маша | 8 | Иванова А. | +7 900 … | Суббота | — | Новая |

Администратор меняет «Статус» вручную: `Новая` → `Позвонили` → `Записаны` / `Отказ`

## 6. Финальный чеклист

### Перед деплоем — обновить в коде

- [ ] Телефон клуба в `public/index.html` (раздел футер)
- [ ] Ссылка на Telegram клуба `@resolute_cheer` → ваша реальная
- [ ] Email для mailto-fallback в `public/js/form.js`
- [ ] Расписание тренировок в `public/index.html`
- [ ] Фото тренеров: заменить `public/assets/images/coaches/coach1-3.jpg`
  на реальные портреты каждого тренера

### После деплоя — проверить

- [ ] Сайт открывается по адресу
- [ ] Сайт корректно отображается на телефоне (iOS Safari + Android Chrome)
- [ ] Меню-бургер открывается и закрывается
- [ ] Галерея открывает лайтбокс, работает свайп
- [ ] Форма показывает ошибки при пустых полях
- [ ] Форма отправляется → пришло сообщение в Telegram
- [ ] Форма отправляется → появилась строка в Google Sheets
- [ ] Отображается экран успеха после отправки

---

*Если что-то не работает — напишите в поддержку Netlify (чат на сайте)
или в поддержку хостинга. Для Telegram Bot проблем — проверьте токен
через getUpdates как описано в шаге 4.*
