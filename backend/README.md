# RESOLUTE — Backend & Интеграции

## Нужен ли бэкенд?

**Нет** — для сайта с только формой заявки полноценный бэкенд (Next.js, Node.js) избыточен.

Используется бессерверный подход:
- **Telegram Bot API** — прямой вызов из браузера, мгновенные уведомления
- **Google Apps Script** — бесплатный FaaS от Google, пишет прямо в Google Sheets

```
Пользователь заполняет форму
         │
         ▼
   public/js/form.js
         │
    ┌────┴────┐
    │         │
    ▼         ▼
Telegram   Google
  Bot      Sheets
(уведом.) (хранение)
```

## 1. Настройка Telegram Bot

### Шаг 1 — Создать бота
1. Напишите в Telegram: `@BotFather`
2. Отправьте команду: `/newbot`
3. Придумайте имя: `Resolute Club Bot`
4. Придумайте username: `resolute_club_bot`
5. Скопируйте **Bot Token**: (формат: `цифры:буквы_и_цифры`)

### Шаг 2 — Получить Chat ID (куда приходят заявки)

**Вариант А — личные сообщения:**
1. Напишите боту любое сообщение
2. Откройте в браузере:
   ```
   https://api.telegram.org/bot<ВАШ_TOKEN>/getUpdates
   ```
3. В ответе найдите `"chat":{"id": XXXXXXXXX}` — это ваш Chat ID

**Вариант Б — группа или канал (рекомендуется):**
1. Создайте группу "RESOLUTE Заявки"
2. Добавьте бота в группу
3. Сделайте бота администратором
4. Откройте `getUpdates` как выше — Chat ID будет отрицательным `-100XXXXXXXXX`

### Шаг 3 — Прописать в коде
Откройте `public/js/form.js`, найдите `CONFIG`:
```js
const CONFIG = {
  TELEGRAM_BOT_TOKEN: '<ВАШ_ТОКЕН_ОТ_BOTFATHER>',
  TELEGRAM_CHAT_ID:   '<ВАШ_CHAT_ID>',
  MODE: 'both',
};
```

### Проверка
Откройте в браузере (замените TOKEN и CHAT_ID):
```
https://api.telegram.org/bot<TOKEN>/sendMessage?chat_id=<CHAT_ID>&text=Тест
```
Должно прийти сообщение "Тест".

## 2. Настройка Google Sheets

### Шаг 1 — Создать таблицу
1. Откройте [Google Sheets](https://sheets.google.com)
2. Создайте новую таблицу: `RESOLUTE — Заявки`

### Шаг 2 — Создать скрипт
1. В таблице: **Расширения → Apps Script**
2. Удалите весь код в редакторе
3. Скопируйте и вставьте содержимое файла `backend/google-apps-script.gs`
4. Нажмите 💾 Сохранить

### Шаг 3 — Развернуть как Web App
1. Нажмите **Развернуть → Новое развёртывание**
2. Тип: **Веб-приложение**
3. Выполнять от имени: **Я (ваш аккаунт)**
4. Кто имеет доступ: **Все** *(анонимный доступ нужен для приёма POST)*
5. Нажмите **Развернуть**
6. При первом деплое — разрешите доступ к таблицам
7. Скопируйте **URL веб-приложения**: `https://script.google.com/macros/s/.../exec`

### Шаг 4 — Прописать в коде
```js
const CONFIG = {
  SHEETS_WEBHOOK_URL: 'https://script.google.com/macros/s/ВАШ_ID/exec',
};
```

### Результат в таблице
Каждая заявка = новая строка:

| Дата/Время | Имя ребёнка | Возраст | Родитель | Телефон | Время | Коммент | Статус |
|---|---|---|---|---|---|---|---|
| 19.05.2025 14:30 | Маша | 8 | Иванова А. | +7... | Пн/Ср/Пт | — | Новая |

**Статусы для администратора:** `Новая` → `Позвонили` → `Записаны` / `Отказ`

## 3. Файл form.js — режимы работы

| MODE | Поведение |
|------|-----------|
| `'both'` | Telegram + Google Sheets (рекомендуется) |
| `'telegram'` | Только Telegram |
| `'sheets'` | Только Google Sheets |
| `'mailto'` | Открывает почтовый клиент (без API) |

## 4. Деплой фронтенда

### Вариант 1 — Netlify (бесплатно, рекомендуется)
```bash
# Установите Netlify CLI
npm install -g netlify-cli

# Деплой из папки public/
netlify deploy --prod --dir=public
```
Или перетащите папку `public/` на [netlify.com/drop](https://netlify.com/drop)

### Вариант 2 — GitHub Pages (бесплатно)
1. Создайте репозиторий на GitHub
2. Скопируйте содержимое `public/` в корень
3. Settings → Pages → Source: main branch

### Вариант 3 — Свой хостинг (Beget, TimeWeb и др.)
- Загрузите содержимое `public/` в `public_html/` через FTP/FileManager
- Стоимость: от 99₽/месяц
